一个简单的demo
1.先编写一个http程序，暴露8000端口，对外提供服务
2.书写Dockerfile文件，构建镜像。
    测试镜像是否成功，可以先docker启动，然后访问看看正不正常
    docker run  -p 8000:8000 wsjtest  把容器端口映射到电脑端口 前面的是电脑的端口
    本机访问不到就去docker里面看看能不能访问。我之前访问不了是把.0.0.0.0写成了127.0.0.1，导致容器内部能访问，容器外面访问不了。
    进入容器命令： docker exec -it 容器编号 /bin/bash
3.书写k8s的deployment和service
    这边deployment创建两个副本，service代理pod的8000端口，并k8s内部也开放8000。
    最后service把自己的8000端口映射到主机的30880
4.运行deploymet.sh 启动服务。
5.访问http://192.168.1.102:30880 （127.0.0.1:30880我这边不知道为什么访问不了，只能用本机ip）
